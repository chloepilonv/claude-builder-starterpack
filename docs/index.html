What it does
Structured pipeline, not chaos

Every product follows the same 9 phases. The orchestrator tracks state, enforces approval gates between phases, and never auto-advances without your go-ahead.

Specialized subagents per phase

Each phase delegates to a purpose-built agent — a spec interviewer, a feasibility planner, a build executor, a design system applier, a deploy debugger. Right tool, right context.

MVP phases defined by your plan

Build phases 3–5 aren't assumed. The plan agent defines exactly what to build in each sprint based on your actual product — so goals come from your spec, not from guessing.

Hooks that enforce quality gates

Git snapshots before every phase. Progress reporter on every tool call. Artifact gate blocks session close if a phase output is missing. Security guard before deploy.

How it works
01
Run /cbs:app-builder in your project directory

The orchestrator checks for .app-builder/state.json. New project: starts at Phase 1. Existing project: shows pipeline status and asks where to resume.

02
Each phase runs in its own subagent

The orchestrator delegates to the right specialist — spec-agent interviews you and produces a rigorous product spec, plan-agent does feasibility and risk, build-agent executes MVP sprints based on what was planned.

03
Approval gate after every phase

When a phase completes, the pipeline status table is rendered and you choose: yes to proceed, review to read the output, redo to re-run, skip, or stop to pause.

04
Jump in anywhere with shortcuts

Don't need to start from scratch — /cbs:build 2 jumps to MVP phase 2, /cbs:design dark brutalist goes straight to design, /cbs:cleanup security runs a security audit anytime.

Agents
CommandAgentWhat it does
/cbs:app-builderOrchestratorManages the full pipeline, tracks state, enforces approval gates
/cbs:specspec-agentInterviews you and produces a rigorous product spec
/cbs:planplan-agentFeasibility check, risk matrix, defines your custom MVP phases
/cbs:build [N]build-agentExecutes any MVP phase per the plan — zero assumed structure
/cbs:frontendfrontend-agentBuilds the functional UI (functionality first, no design polish yet)
/cbs:design [style]design-agentApplies a full design system from a style argument or moodboard
/cbs:docsdocs-agentWrites README, ARCHITECTURE, API docs; cleans the repo
/cbs:deploy [platform]deploy-agentCI/CD setup, production deploy, smoke tests, debugging
/cbs:cleanup [scope]cleanup-agentDead files, secrets audit, dep patches, README check — run anytime
/cbs:new-feature [desc]feature-agentMini spec + build + test + deploy for any post-launch feature
/cbs:nebius [action]nebius-agentNebius AI Cloud — GPU instances, jobs, storage, cost tracking
Installation
01

Clone the repo

git clone https://github.com/chloepilonv/claude-builder-starterpack ~/claude-builder-starterpack
02

Register the marketplace — run inside Claude Code

/plugin marketplace add ~/claude-builder-starterpack
03

Install the plugin — run inside Claude Code

/plugin install claude-builder-starterpack@claude-builder-starterpack
04

Restart Claude Code, then in any project directory:

/cbs:app-builder
